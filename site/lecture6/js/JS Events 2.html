Say hello to

<button id="btn" onclick="sayHello()">Say Hello</button> <script> function sayHello() { console.log(this); // this 指向 window 對象 console.log(this.id); // this.id 為 undefined } </script>

在這個例子中，我們在 <button> 元素上使用了 onclick 屬性，並指定了 sayHello 這個函數作為事件處理器。 當我們點擊這個按鈕的時候，sayHello 函數會被執行，但是在這個函數裡面，this 並不是指向 <button> 元素，而是指向 window 對象，這是因為 sayHello 函數是在全域範圍裡定義的，所以它的 this 也是全域的。這樣就會造成一些混亂， 因為我們可能想要在事件處理器裡面使用 <button> 元素的一些屬性或方法，比如說它的 id 或 style，但是這些都會變成 undefined， 因為 window 對象並沒有這些屬性或方法。

如果我們想要讓事件處理器的 this 指向 <button> 元素，我們就可以使用非侵入式事件綁定的方法，也就是使用 addEventListener 方法，像這樣：

<button id="btn">Say Hello</button> <script> let btn = document.querySelector("#btn"); // 選取按鈕元素 btn.addEventListener("click", sayHello); // 綁定事件處理器 function sayHello() { console.log(this); // this 指向按鈕元素 console.log(this.id); // this.id 為 "btn" } </script>

在這個例子中，我們先用 document.querySelector 方法選取了按鈕元素， 然後用 addEventListener 方法將 sayHello 函數綁定到按鈕元素的 click 事件上。 這樣做的好處是，當我們點擊按鈕的時候，sayHello 函數裡面的 this 就會指向按鈕元素， 而不是 window 對象，這樣我們就可以正常地使用按鈕元素的屬性或方法，比如說它的 id 或 style。這樣就可以避免一些混亂和錯誤。
